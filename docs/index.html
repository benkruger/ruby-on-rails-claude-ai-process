FLOW

An opinionated 6-phase software development lifecycle for Claude Code. Every feature follows the same phases in the same order. No shortcuts. Supports Rails and Python.

Why

Claude Code is powerful, but undisciplined by default. Left to itself, it writes code, skips tests, misses edge cases, and moves on. FLOW imposes structure. Not bureaucracy — discipline.

FLOW is for the engineer who wants to stay in the driver’s seat: who decides what gets built, approves the design, reviews the plan, and merges the result. You control the autonomy — fully manual, fully autonomous, or anywhere in between, per skill. You remain the engineer. Claude does the work.

How

Every feature goes through 6 phases. Each phase must complete before the next one starts. If something was missed, go back.

1
Start Branch, PR, green CI baseline, dependencies upgraded
2
Plan Explore the codebase, design the approach, identify risks, write tasks
3
Code Test first, then implement, review every diff, commit per task
4
Code Review Three lenses on the same diff — clarity, correctness, and safety
5
Learn Capture what went wrong, route corrections to permanent homes
6
Complete Merge PR, clean up, done

The Learning System

Most AI coding tools forget everything when the session ends. FLOW captures corrections as they happen, rescues session memory before worktrees are destroyed, and routes each learning to the right permanent home — where it compounds across every future feature.

1
Capture User corrects Claude → /flow-note captures it automatically
2
Observe Claude writes patterns and observations to auto-memory during feature work
3
Synthesize Learn reads three sources: CLAUDE.md rules · conversation context · state/plan data
4
Route Each learning is routed to its correct permanent destination
5
Survive Worktree is destroyed — nothing is lost

5 Destinations

Global CLAUDE.md

Process rules that apply to all projects. “Always run bin/ci before committing.”

Project CLAUDE.md

Project-specific architecture. “Callbacks set fields from Current — don’t pass explicitly.”

Global Rules

Universal coding standards. “Never use guard clauses.”

Project Rules

Project-specific gotchas. “expect_worker uses lambdas, not kind_of().”

Project Memory

Patterns and observations. “Tests with Time.zone.now fail near midnight.”

Why FLOW Is Different

🎛️
Zero to YOLO

Fully manual, fully autonomous, or anywhere in between — configured per skill. Perfect for engineers still finding their comfort level with AI automation.

🧠
Mistakes become patterns

Correct Claude once and the learning system routes it to CLAUDE.md, rules, or memory. It never makes the same mistake twice.

⚡
Latest Claude Code features

Plan mode for exploration, /simplify for refactoring, Opus 4.6 for reasoning, model frontmatter for automatic selection. Built on the platform, not around it.

💾
Survives everything

Compaction, /clear, new terminal, a week away. State file plus session hook means Claude picks up exactly where it left off.

🪶
Zero footprint

Zero dependencies. Zero permanent artifacts in your repo. One .flow-states file while you work — deleted at cleanup.

🔄
Always current

Marketplace plugin — updates ship automatically. No manual upgrades, no version chasing.

🌐
Any Framework

Data-driven framework support. Rails and Python included. Add your own.

Get Started

All 6 phase skills, state file infrastructure, session hooks, and reference documentation in one plugin.

Step 1: Install in Claude Code

/plugin marketplace add benkruger/flow
/plugin install flow@flow-marketplace

Step 2: Initialize in your project (once per project)

/flow-prime

Step 3: Start a feature

/flow-start invoice pdf export
